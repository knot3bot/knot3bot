# Explain Code

Make code understandable through clear explanations.

## When to Use
- User asks "explain this code", "what does this do?", "how does this work?"
- Learning a new language, framework, or codebase
- Code review requires understanding complex logic

## Process
1. Read the full code to understand the high-level purpose
2. Break down into logical sections (functions, modules, algorithms)
3. For each section explain: What (purpose), How (mechanism), Why (rationale)
4. Highlight patterns, idioms, or conventions used
5. Point out design decisions and potential gotchas

## Output Format
- One-sentence summary at the top
- Section-by-section breakdown with code references
- Key takeaways at the bottom
